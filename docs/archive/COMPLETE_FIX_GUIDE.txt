WHAT WAS THE PROBLEM?
─────────────────────────────────────────────────────────────────────────────

Your website showed: "500: INTERNAL_SERVER_ERROR - FUNCTION_INVOCATION_FAILED"

Root Cause:
  • Vercel couldn't find the entry point for your serverless function
  • The Express app existed but wasn't properly exposed to Vercel
  • Vercel's routing configuration was incomplete

Why It Crashed:
  • Vercel tried to execute the function
  • Function had no handler (api/index.ts didn't exist)
  • Function initialization failed
  • Vercel returned "FUNCTION_INVOCATION_FAILED" error

STEP 1: Created api/index.ts
─────────────────────────────────────────────────────────────────────────────

What It Does:
  • Acts as the main entry point for Vercel serverless functions
  • Imports the compiled Express app from dist/app.js
  • Receives requests from Vercel router
  • Passes them to Express for handling
  • Returns responses back to clients

Code Structure:
  import { VercelRequest, VercelResponse } from '@vercel/node';
  
  export default async (req, res) => {
    const { default: app } = await import('../dist/app');
    return app(req, res);  // Pass to Express
  };

Why It Works:
  • Vercel recognizes api/index.ts as a serverless function
  • Uses @vercel/node for proper request/response types
  • Dynamically imports compiled app (guarantees latest code)
  • Handles errors gracefully

STEP 2: Updated vercel.json
─────────────────────────────────────────────────────────────────────────────

Old Configuration (Broken):
  {
    "buildCommand": "npm run build",
    "outputDirectory": "dist"
  }
  └─ Missing: Tells Vercel WHAT to run and HOW to route

New Configuration (Fixed):
  {
    "buildCommand": "npm run build",
    "installCommand": "npm install",
    "outputDirectory": "dist",
    "functions": {
      "api/index.ts": {
        "runtime": "nodejs20.x",
        "memory": 1024,
        "maxDuration": 30,
        "includeFiles": "dist/**"
      }
    }
  }

What It Does Now:
  • runtime: nodejs20.x → Uses Node.js 20 (matches your package.json)
  • memory: 1024 → Allocates 1GB RAM (plenty for your app)
  • maxDuration: 30 → Function can run for 30 seconds (Vercel limit)
  • includeFiles: Packages the dist/ folder with the function

─────────────────────────────────────────────────────────────────────────────

Why It's Needed:
  • Provides VercelRequest and VercelResponse types
  • Adapts Node.js HTTP to Vercel's serverless format
  • Ensures compatibility between Express and Vercel

Added To package.json:
  "dependencies": {
    "@vercel/node": "^3.1.0",
    ...other packages...
  }

Auto-Installed: During deployment when Vercel runs "npm install"

STEP 4: Error Handling (Already Done)
─────────────────────────────────────────────────────────────────────────────

From Previous Session (All 3 Layers):

Layer 1: Route Handlers (asyncHandler)
  router.post('/login', asyncHandler(loginHandler));
  ├─ Catches errors in async functions
  ├─ Prevents unhandled promise rejections
  └─ Passes errors to Express error middleware

Layer 2: Express Error Middleware (errorHandler)
  app.use((err, req, res, next) => {
    return res.status(err.status || 500).json({...});
  });
  ├─ Catches all errors from routes
  ├─ Returns JSON error response
  └─ Prevents HTML 500 page

Layer 3: Process Error Handlers (server.ts)
  process.on('unhandledRejection', handler);
  process.on('uncaughtException', handler);
  ├─ Catches process-level errors
  ├─ Logs for debugging
  └─ Prevents complete function crash

                           HOW IT WORKS NOW
═══════════════════════════════════════════════════════════════════════════════

Request Flow:
───────────────

  Client Request
      ↓
  Vercel Router (detects /api/...)
      ↓
  Invokes: api/index.ts handler
      ↓
  Handler imports: dist/app.js (Express)
      ↓
  Express processes request through middleware:
    • cors middleware
    • helmet security
    • json parser
    • authentication
    • route matching
      ↓
  Route Handler (wrapped in asyncHandler):
    ├─ Executes async logic
    ├─ If error thrown:
    │   ├─ asyncHandler catches it
    │   └─ Passes to error middleware
    ├─ If success:
    │   └─ Returns JSON response
      ↓
  Express Error Middleware (if error occurred):
    ├─ Receives error
    ├─ Formats JSON response
    ├─ Sets HTTP status (400, 401, 500, etc)
    └─ Sends to client
      ↓
  Vercel returns response to client
      ↓
  Client receives JSON (not crash page!)
